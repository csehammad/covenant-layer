# Contributing

Thanks for contributing.

This project is trying to define a real model and protocol, not just a nice essay. Precision matters more than volume.

## Useful contributions

Useful contributions include:

- tighter role definitions
- clearer state transitions
- better example flows
- missing edge cases
- security or trust model corrections
- framework feedback
- terminology cleanup

## Less useful contributions

Less useful contributions include:

- vague hype
- generic AI futurism
- very broad scope expansion
- provider internals that do not affect interoperability
- replacing precise language with slogan-heavy language

## Before opening a PR

Read these first:

- `README.md`
- `docs/mental-model.md`
- `spec/01-problem.md`
- `spec/02-core-protocol.md`
- `spec/03-security.md`

If your proposal changes the model or protocol, explain:

1. what problem it solves
2. what state transitions it affects
3. what risks it introduces
4. how an independent implementer would read it

## Writing style

Write plainly.

Prefer:
- short sentences
- direct definitions
- explicit tradeoffs
- clean non-goals

Avoid:
- inflated language
- marketing tone
- vague “future of AI” language
- unnecessary jargon
