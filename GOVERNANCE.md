# Governance

Covenant Layer is currently editor-led.

That means the repository is open to discussion and contributions, but the maintainers curate the core model, vocabulary, and protocol structure while the work is still unstable.

## Current goals

The near-term goals are:

- define the problem clearly
- make the protocol lifecycle precise
- build a small demo
- sharpen the security model
- stabilize the core vocabulary

## Decision principles

Changes should improve one or more of the following:

- clarity
- interoperability
- safety
- auditability
- implementability

Changes should be rejected if they:

- blur the difference between layer, network, and protocol
- move provider internals into the core spec
- depend on one vendor platform
- turn the project into generic workflow tooling
- weaken authority, evidence, settlement, or dispute semantics

## Future governance

If the project gains real implementers, governance should evolve toward:

- versioned drafts
- public design review
- clear extension rules
- interop testing
- a standards-track path
