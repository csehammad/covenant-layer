# Travel demo

## User objective

"Get me to Paris from New York next Thursday. Prefer nonstop if the price difference is reasonable. No overnight layovers. Business class only if it stays under $2,500. Show me the best two before booking."

## Fake providers

### Provider A
- nonstop
- higher price
- short hold
- moderate refund terms

### Provider B
- one stop
- lower price
- better refund terms

### Provider C
- business class
- near budget limit
- tighter change policy

## Demo flow

1. User enters objective
2. Agent receives authority grant
3. Broker routes objective
4. Providers return offers
5. Agent compares offers
6. User approves one
7. Provider fulfills
8. Provider submits proof
9. Settlement is recorded

## Success condition

The viewer should come away with one clear understanding:

The agent is not just calling tools.
It is coordinating around commitments.
